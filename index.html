PL/SQL Profiler - Project Overview

Executive Summary

The PL/SQL Profiler is a comprehensive, enterprise-grade performance analysis platform designed for Oracle PL/SQL applications. This tool provides deep insights into PL/SQL code execution, helping developers and DBAs identify performance bottlenecks, optimize database operations, and ensure code quality through advanced static analysis.

üéØ Core Purpose

  • Performance Analysis: Line-by-line execution profiling of PL/SQL code
  • Code Quality: Static analysis with configurable rules and recommendations
  • Enterprise Integration: Support for complex business logic and real-world scenarios
  • Hierarchical Profiling: Advanced call tree analysis using Oracle's DBMS_HPROF
  • Web-based Interface: Modern, intuitive dashboard for analysis and reporting

üèóÔ∏è Architecture Overview

Core Components

  1. Profiling Engine (plsql_profiler.core)
    • PLSQLProfiler: Main profiler using Oracle's DBMS_PROFILER
    • HierarchicalProfiler: Advanced profiling with DBMS_HPROF
    • MockPLSQLProfiler: Testing and demo mode without Oracle connection
  2. Static Analysis Engine (static_analyzer.py)
    • Rule-based code analysis with 15+ built-in rules
    • Configurable severity levels and custom parameters
    • Real-time code quality scoring and recommendations
  3. Web Interface (plsql_profiler.web)
    • Flask-based modern web application
    • Interactive dashboards and visualizations
    • Real-time profiling control and monitoring
  4. License Management (license_manager.py)
    • Enterprise licensing with expiration tracking
    • Usage monitoring and compliance reporting

üåü Key Features

1. Real-time PL/SQL Profiling

  • Line-Level Performance Analysis
    • Execution count per line
    • Total, minimum, and maximum execution times
    • Resource consumption tracking (CPU, I/O, memory)
    • SQL statement analysis within PL/SQL blocks
  • Advanced Metrics
    • Buffer gets and disk reads
    • Rows processed and affected
    • Wait event analysis
    • Bind variable capture and analysis

2. Hierarchical Call Tree Analysis

  • DBMS_HPROF Integration
    • Complete call stack visualization
    • Function call relationships and dependencies
    • Time distribution across call hierarchy
    • Performance bottleneck identification
  • Fallback Mechanism
    • Smart fallback for Oracle XE limitations
    • Synthetic call tree generation
    • Maintains analysis capability across Oracle versions

3. Static Code Analysis

  • Comprehensive Rule Engine
    • Security Rules: SQL injection detection, dynamic SQL validation
    • Performance Rules: SELECT * detection, nested loops, commit-in-loop
    • Maintainability Rules: Magic numbers, hardcoded values, long procedures
    • Reliability Rules: Exception handling, cursor management
    • Code Quality Rules: Implicit conversions, bind variable usage
  • Configurable Analysis
    • YAML-based rule configuration
    • Custom severity levels (Info, Warning, Error, Critical)
    • Rule-specific parameters and thresholds
    • Runtime rule updates and reloading

4. Enterprise Database Integration

  • Real-world E-commerce Scenarios
    • Customer management with tier-based processing

    • Financial calculations with tax and discount handling
  • Sample Procedures for Testing

    • ec_analyze_customer_behavior: Customer analytics and upgrades
    • ec_restock_low_inventory: Automated inventory management

5. Modern Web Interface

  • Dashboard Features
    • Real-time profiling status and controls
    • Interactive performance charts and graphs
    • Run history with filtering and search
    • Export capabilities for reports and data
  • Analysis Views
    • Detailed run analysis with drill-down capabilities

    • Call tree visualization with performance metrics
    • Code quality reports with actionable recommendations
  • Settings and Configuration
    • Rule configuration interface
    • Database connection management
    • License status and usage tracking
    • System diagnostics and health checks

6. Multi-Mode Operation

  • Production Mode
    • Full Oracle database integration
    • Complete profiling capabilities
    • Enterprise feature set
    • License validation and compliance
  • Mock Mode
    • Offline development and testing

üõ†Ô∏è Technical Specifications

Database Requirements

  • Oracle Database: 11g or higher (tested with Oracle XE)
  • Required Packages: DBMS_PROFILER, DBMS_HPROF (when available)
  • Privileges: CREATE TABLE, SELECT on V$ views, EXECUTE on DBMS packages

System Requirements

  • Python: 3.8+ with pip
  • Dependencies: Flask, oracledb, click, pyyaml, jinja2
  • Storage: Minimal (profiling data stored in Oracle)
  • Network: HTTP access for web interface (default port 5000/5003)

Deployment Options

  • Docker: Containerized deployment with docker-compose
  • Standalone: Direct Python installation with virtual environment
  • Development: Local setup with mock mode for testing
